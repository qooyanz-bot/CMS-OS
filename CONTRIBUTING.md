# CMS-OSへのコントリビューション

## README更新時の翻訳ルール

README.mdを変更する場合は、同じ変更内容を次のREADMEにも反映します。

- README.en.md
- README.zh-CN.md
- README.es.md
- README.ko.md
- README.de.md
- README.fr.md

日本語版READMEを正本とし、翻訳の意味、機能名、API/MCPの操作名、ロードマップの内容を一致させます。翻訳だけを先行して変更する場合は、変更理由をPull Requestに記載してください。

## API/MCPファーストの原則

CMS-OSの機能は、管理画面だけに実装してはいけません。新しい操作を追加する場合は、次の順序で設計します。

1. ドメイン操作と権限を定義する
2. バージョン付きAPI契約を定義する
3. MCPツールから同じドメイン操作を呼び出せるようにする
4. 管理画面はAPI/MCPを利用するクライアントとして実装する
5. OpenAPI、MCPツール定義、READMEの説明を更新する

APIとMCPで異なる業務ロジックを実装せず、同じドメインサービス、認証、テナント境界、監査ログ、承認ルールを共有します。
