# Portal Planning Agent 実行結果

対象: `CMS-OS`

## 前提

- このリポジトリには、現時点でのページ一覧、Search Console の実データ、既存カテゴリ構成がありません。
- そのため、以下は `CMS-OS` の方針と公的ソースを元にした、初回の仮説ベースのポータル企画です。
- 目的は「新しい業界テーマの発掘」「利用者の検索意図整理」「不足している地域・テーマの補完」「新規ページの作成提案」を、CMS-OS のコンテンツ運用に落とし込むことです。

## 1. 新しい業界テーマの提案

| 優先 | テーマ | ねらい | 代表的な検索意図 |
|---|---|---|---|
| 1 | 生成AI・業務改革 | 需要が広く、導入前後の検索が多い | 何ができるか、比較、費用、導入手順、社内ルール |
| 2 | 人手不足・省人化 | 企業課題として強く、業種横断で刺さる | 人手不足対策、採用難、外注、RPA、ロボット、代替策 |
| 3 | 地域観光・インバウンド | 地域ページと相性がよく、回遊導線を作りやすい | 観光客動向、モデルコース、季節、アクセス、宿泊 |
| 4 | モビリティDX・SDV | 技術・製造・サプライチェーン文脈で伸ばしやすい | SDVとは、導入事例、競合比較、開発体制 |
| 5 | GX・省エネ・資源循環 | 補助金、規制、実務ノウハウの検索が多い | 脱炭素、補助金、CO2削減、再エネ、循環型経営 |
| 6 | 地方創生・移住・空き家再生 | 地域課題と生活者検索の両方を拾える | 移住先比較、住みやすさ、空き家活用、支援制度 |

### 採用理由

- `CMS-OS` は「業界」「地域」「役割」「課題」でコンテンツを分ける設計と相性がよいです。
- 上記テーマは、単なる情報記事ではなく、比較・導入・制度確認までページ群を広げやすいです。
- 人手不足、地域活性、観光、GX は、公的統計や制度変更で更新性を保ちやすく、継続運用に向いています。

## 2. 利用者の検索意図分析

### A. 理解したい

- 例: 「生成AIとは」「SDVとは」「地域観光DXとは」
- 必要なページ: 定義、全体像、図解、用語集、FAQ
- 作るべき導線: トップ説明 -> 関連テーマ -> 事例 -> 比較

### B. 比べたい

- 例: 「AとBの違い」「どの制度が得か」「どの地域が有利か」
- 必要なページ: 比較表、選定基準、メリット・デメリット、判断フロー
- 作るべき導線: 比較ページ -> ケース別診断 -> 申込・問い合わせ

### C. 実行したい

- 例: 「導入手順」「申請方法」「チェックリスト」「テンプレート」
- 必要なページ: 手順書、チェックリスト、サンプル、テンプレート、入力例
- 作るべき導線: 実行手順 -> 必要資料 -> 次のアクション

### D. 確認したい

- 例: 「費用」「補助金」「法規制」「リスク」「失敗例」
- 必要なページ: 費用相場、注意点、よくある失敗、制度解説、リスク一覧
- 作るべき導線: 注意点 -> 対策 -> 相談窓口

### E. 地域で絞りたい

- 例: 「北海道 観光」「福岡 人手不足」「東北 移住」
- 必要なページ: 地域ハブ、都道府県ページ、都市ページ、地域別事例
- 作るべき導線: 地域トップ -> 県別ページ -> 事例 -> 関連テーマ

## 3. 不足している地域・テーマの発見

### 地域の不足

- 首都圏や大都市だけでなく、都道府県単位の受け皿を作る必要があります。
- 特に、北海道、東北、北陸、中国、四国、九州・沖縄の地域ページは、後回しにされやすい一方で需要があります。
- さらに、県庁所在地だけでなく、第二都市・観光拠点・工業都市・港湾都市まで広げると検索取りこぼしを減らせます。

### テーマの不足

- 人手不足対策は、採用記事だけでは足りません。省人化、業務分解、委託、ロボット、AI活用まで必要です。
- 観光テーマは、一般観光だけではなく、インバウンド、DMO、広域周遊、宿泊、交通、観光消費まで必要です。
- 地方創生テーマは、移住だけではなく、空き家、教育、医療、交通、仕事、生活コストを含めるべきです。
- GXテーマは、理念記事だけでは弱く、補助金、設備更新、実装手順、投資回収まで必要です。
- モビリティDXは、技術記事だけでなく、サプライチェーン、法規、運用、保守、実証まで必要です。

### 優先して埋めるべき空白

1. 地域別のハブページ
2. 課題別のハブページ
3. 比較・選定ページ
4. 制度・補助金ページ
5. 事例・失敗例ページ

## 4. 新規ページの作成提案

### まず作るべきページ群

- `業界ハブページ`
  - 例: 生成AI、観光、GX、モビリティ、地方創生
- `地域ハブページ`
  - 例: 都道府県、地方ブロック、主要都市
- `課題解決ページ`
  - 例: 人手不足、売上拡大、省エネ、集客、採用
- `比較ページ`
  - 例: サービス比較、制度比較、地域比較、導入方式比較
- `事例ページ`
  - 例: 導入前後、成功例、失敗例、業界別ケース
- `FAQ / 用語集`
  - 例: 初学者向けの理解支援と内部リンクの起点

### CMS-OS への落とし込み

- テーマトップを起点に、地域ページと課題ページを内部リンクで相互接続します。
- 1ページ完結ではなく、`理解 -> 比較 -> 実行 -> 確認` の流れで系列化します。
- 公的統計、制度、更新情報を紐づけて、定期的に更新されるページにします。

## 5. 実行優先度

1. 既存ページ一覧を棚卸しする
2. 既存流入データから検索意図を分類する
3. 業界ハブと地域ハブを先に作る
4. 比較・FAQ・事例を積む
5. 制度改定や統計更新を定期反映する
