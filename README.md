# CMS-OS

AIエージェントネイティブな企業コンテンツ管理基盤です。

CMS-OSは、企業情報・採用・PR・IR・Blogを一つの基盤で管理し、AIエージェントによる企画、提案、下書き、事実確認、清書、SEO最適化、静的公開までを支援します。

現在はOSSとして開発を開始した初期段階です。

## CMS-OSが目指すもの

従来のCMSは、人間が記事を入力・編集・公開するための道具でした。

CMS-OSは、企業の検証済み情報とブランド方針を理解したAIエージェントが、コンテンツの目的や対象ポジションに応じて、最適な情報発信を支援するCMSを目指します。

```text
企業情報・ブランド情報
        ↓
AIによる企画・提案
        ↓
構成案・下書き
        ↓
事実確認・SEO最適化
        ↓
清書・承認
        ↓
静的ページ生成・公開
```

## 主な機能

### AIエージェントネイティブ

- コンテンツテーマの提案
- 検索意図・読者・対象ポジションの分析
- 記事構成の作成
- 採用、PR、IR、Blogの下書き生成
- 企業情報に基づく初稿作成
- 誤字脱字・表記ゆれ・冗長表現の修正
- ブランドトーンに合わせた清書
- 事実情報・数値・日付・出典の確認
- SEOタイトル・メタディスクリプションの生成
- 内部リンク・FAQ・構造化データの提案

AIは自動公開者ではなく、企業の編集チームを支援する編集エージェントとして動作します。IR、法務、給与、代表者情報などの重要情報は、人間の承認を経て公開します。

### ポジション別コンテンツ生成

採用ポジション、読者、業界、地域、コンテンツ目的に応じて、訴求内容や文章構成を変えます。

- 採用ポジション別の求人原稿
- 候補者向けの職種紹介
- メディア向けのプレスリリース
- 投資家・株主向けのIRコンテンツ
- 初心者・導入検討者・既存顧客向けのBlog記事

### 企業コンテンツ管理

- 企業基本情報
- 求人・職種・社員紹介
- プレスリリース・ニュース・イベント
- IR資料・IRカレンダー・訂正履歴
- Blog記事・著者・シリーズ・タグ
- 画像・動画・PDF・ブランド資産
- 多言語・公開範囲・有効期限
- バージョン履歴・承認履歴・監査ログ

### SEOファースト

SEOを公開後の追加作業ではなく、企画・執筆・編集・公開の全工程に組み込みます。

- 検索意図に基づく企画提案
- SEOタイトル・見出し・ディスクリプション
- canonical、robots、sitemapの管理
- OGP・パンくず・内部リンク
- Article、BlogPosting、JobPosting、FAQなどの構造化データ
- 重複コンテンツ・リンク切れ・孤立ページの検出
- 著者、出典、公開日、更新日の管理
- AI検索で引用されやすい要約・FAQ・比較表
- 画像alt属性、画像最適化、表示速度の検査

### Cloudflare Pages向け静的公開

CMS-OSは、公開サイトを静的HTMLとして生成し、Cloudflare Pagesへ配信できる構成を採用します。

```text
CMS-OS
  ↓ 承認済みコンテンツ
静的HTML・CSS・JS・画像・JSON-LD
  ↓
BuilderOS Adapter
  ↓
Cloudflare Pages
```

管理画面、CMS API、AI処理はCMS基盤側で実行し、公開ページは静的ファイルとして配信します。これにより、表示速度、可用性、SEO、運用コストの最適化を目指します。

Cloudflare Pagesの静的アセット配信仕様については、以下の公式ドキュメントを参照してください。

- [Cloudflare Pages Functionsの料金](https://developers.cloudflare.com/pages/functions/pricing/)
- [Cloudflare Pagesの制限](https://developers.cloudflare.com/pages/platform/limits/)
- [Cloudflare Pagesの静的HTML公開](https://developers.cloudflare.com/pages/framework-guides/deploy-anything/)

## AIエージェントの役割

CMS-OSでは、役割ごとにAIエージェントを分けます。

| エージェント | 主な役割 |
|---|---|
| 企画エージェント | テーマ、検索意図、対象読者、構成の提案 |
| 下書きエージェント | 企業情報に基づく本文の初稿作成 |
| 事実確認エージェント | 出典、数値、日付、企業情報の確認 |
| 清書エージェント | 日本語、表記、ブランドトーン、読みやすさの調整 |
| SEOエージェント | 検索意図、構造化データ、内部リンクの最適化 |
| 公開エージェント | 承認済みコンテンツのビルド・公開準備 |

エージェントの実行履歴、参照データ、生成内容、承認者を保存し、生成過程を追跡可能にします。

## コンテンツワークフロー

```text
REQUESTED
  ↓
PROPOSED
  ↓
DRAFTED
  ↓
FACT_CHECKED
  ↓
SEO_REVIEWED
  ↓
EDITED
  ↓
APPROVED
  ↓
PUBLISHED
```

公開済みコンテンツは上書きせず、バージョンとして管理します。特にIRコンテンツでは、訂正・撤回前の内容も監査可能な状態で保持します。

## 想定アーキテクチャ

- CMS管理画面：Next.js、React、TypeScript
- 編集エンジン：Tiptap
- データベース：PostgreSQL
- 認証・権限：Auth基盤＋行単位のアクセス制御
- AI連携：複数モデルに対応できるプロバイダー抽象化
- 非同期処理：キュー・ワークフロー基盤
- メディア：オブジェクトストレージ
- 公開：静的HTML生成＋Cloudflare Pages
- 外部連携：BuilderOS Adapter

CMS、AI、ワークフロー、メディア、公開処理をモジュールとして分離し、初期はモジュラーモノリスとして開発します。

## OSSとしての開発方針

CMS-OSは、企業コンテンツの作成・承認・公開・再利用に関する基盤をOSSとして公開し、開発者、企業、編集者、採用担当者、広報担当者、IR担当者が改善に参加できるプロジェクトを目指します。

重視する原則は次の通りです。

- 企業の事実情報を尊重する
- AI生成物を検証可能にする
- 人間の承認と監査可能性を確保する
- SEOとアクセシビリティを標準機能にする
- 静的公開による高速・低コストな運用を実現する
- 特定ベンダーに依存しない拡張可能な設計にする
